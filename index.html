Comment ça “vite fait, bien fait” ?

Critères rapides à mettre en place même si l'accessibilité est hors scope.

Parce qu'avec des petites choses simples on peut enlever beaucoup de barrières.

Mettre un attribut lang

<html lang="fr">
<p>Texte en français <span lang="en">Some english content</span></p>

Permet aux lecteurs d'écrans de prononcer correctement le contenu du site

Mauvaise prononciation du lecteur d'écran

Mettre un attribut lang

Bonne prononciation du lecteur d'écran

Ne pas utiliser maximum-scale= 1.0

<meta name="viewport" content="width=device-width, initial-scale=1.0 maximum-scale="1.0">

On ne peut plus zoomer la page sur mobile

Ne pas supprimer l’outline

Cela aide à se repérer dans la page notamment dans le cas d'une navigation clavier.

Vous voulez les masquer ?

Masquer des éléments de manière accessible

Utiliser une sémantique cohérente

							<span onclick="btnFunction();" class="btn btn-primary">Cliquez ici</span>

ARIA est votre ami

  • Attributs ARIA et role
  • On peut naviguer d'un role landmark à un autre via un lecteur d'écran ( banner, main,...)
  • Première règle : ne pas utiliser ARIA
  • JavaScript, Ajax,... ARIA à la rescousse !

Bien organiser les niveaux de titres

Très important pour la navigation par titres pour les lecteurs d'écrans !

Après un h1 on met un h2 et non un h3

Utiliser CSS si le h2 doit ressembler à un h3

Mettre un attribut alt sur les images

  • L'image est décorative → alt=""
  • Texte dans l'image → alt="Même texte que dans image"
  • L'image représente une action → alt="Ajouter au panier"
  • L'image est informative → alt="Emplacement"

Détail de l'utilisation de l'attribut alt (en anglais)

Utiliser des vrais label

Le placeholder n'est pas un label, il est là pour donner une suggestion de saisie.

Ne pas s'en servir pour donner une information importante (format de saisie de date par exemple)

Technique de floating labels (en anglais)

À n'utiliser qu'en dernier recours (en anglais) 

							<label for="place">Où préférez-vous vivre ?</label>
<input type="text" id="place" name="place" placeholder="Campagne, Ville">

Comment tester ?

  • Touche tab
  • Désactiver les CSS/JS
  • Passer son code HTML au validateur du W3C
  • Utiliser un lecteur d'écran (NVDA, VoiceOver, Jaws,...)

Attention toutefois

Ces critères sont un premier pas vers un site accessible mais il reste beaucoup à faire !

Il est essentiel de se former pour savoir comment faire un site complètement accessible.
